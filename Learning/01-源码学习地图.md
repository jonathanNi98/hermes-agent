# Hermes Agent 源码学习地图 — 第一阶段

## 一、项目定位

**Hermes Agent** 是 Nous Research 的自改进 AI Agent，主打"内置学习循环"。核心特点：
- 多模型支持（Anthropic、OpenAI、Google、Azure、Gemini、Codex 等）
- 持久化记忆系统（SQLite + embedding）
- 任务拆解（同步子代理 + 异步看板）
- 工具系统（80+ 工具，含 MCP 插件）
- 安全边界（Approval 审批链、路径限制、命令白名单）

```

    │
    ▼
cli.py 或 gateway/run.py
    │
    ▼

    │                                                │

    │                                                │
    ▼                                                │
run_conversation()  ← conversation_loop.py (核心)   │
    │                                                │
    ├── _restore_or_build_system_prompt()           │
    │       └── system_prompt.py                    │
    │                                                │
    ├── prefetch_all()  ← memory_manager.prefetch_all()
    │                                                │
    ├── while not done:                             │
    │       ├── build_messages()                    │
    │       │       ├── system_prompt               │
    │       │       ├── memory_context             │
    │       │       ├── history                    │
    │       │       └── tools_schema               │
    │       │                                       │
    │       ├── _api_call()  ← 模型调用             │
    │       │       └── providers/ (各种 adapter)  │
    │       │                                       │
    │       ├── 解析 tool_calls                     │
    │       │                                       │
    │       ├── tool_executor.py  ← 工具调度        │
    │       │       └── tools/registry.py          │
    │       │                                       │
    │       └── 工具结果填入 messages               │
    │                                                │
    ├── sync_all()  ← memory_manager.sync_all()    │
    │                                                │
    └── return result + 更新 history                │
    │                                                │
    └── 状态持久化 → hermes_state.py + SQLite      │
```

## 八、调用链速查（关键路径）

### 用户消息 → 模型响应

```
run_conversation(user_message)
    │
    ├─► _build_messages()
    │       ├─► system_prompt (system_prompt.py)
    │       ├─► memory_context (memory_manager.prefetch)
    │       └─► tools_schema (model_tools.get_tool_definitions)
    │
    ├─► _api_call_with_retries()
    │       └─► providers/* (根据 runtime 选择)
    │
    ├─► 解析 response.tool_calls[]
    │       └─► tool_executor.execute()
    │               └─► registry.dispatch(name, args)
    │                       └─► tools/<name>.py handler
    │
    └─► memory_manager.sync_all()
```

### 工具注册 → 工具调用

```
tools/<name>.py (模块加载时)
    │
    └─► registry.register(
            name=...,
            schema={...},
            handler=my_func,
            check_fn=availability_check
        )
    │
    ▼
model_tools.get_tool_definitions()
    │
    └─► registry.get_definitions(tool_names)
    │
    ▼
API 返回 tool_call
    │
    ▼
tool_executor.execute(name, args)
    │
    └─► registry.dispatch(name, args)
            └─► handler(args) → JSON string
```

## 九、值得学习的设计模式

| 设计 | 所在文件 | 说明 |
|-----|---------|------|
| 三层 Prompt 架构 | system_prompt.py | stable/context/volatile 分离，缓存友好 |
| Provider 模式 | memory_manager.py | 多后端可替换 |
| 注册中心模式 | registry.py | 工具/工具集集中管理 |
| Frozen Snapshot | memory_tool.py | 写时更新文件，读时用快照 |
| Streaming Scrubber | memory_manager.py | 流式输出中过滤敏感标签 |
| Generation Counter | registry.py | 无锁读并发，mutation 递增 |
| check_fn TTL Cache | registry.py | 30s 缓存避免重复探测 |
| Prefetch/Sync 分离 | memory_manager.py | 预取和回写分开，优化延迟 |
| Tool Executor Guardrails | tool_executor.py | 多层安全检查链 |

## 十、潜在问题、技术债、安全风险

### 潜在问题
- `conversation_loop.py` 4751 行，单一职责过重
- `delegate_tool.py` 2801 行，嵌套逻辑深
- 测试覆盖率不均，核心循环测试少

### 技术债
- 部分 adapter（Azure/Gemini）与主路径重复代码多
- `hermes_state.py` 5000+ 行，混合了太多职责
- 一些工具的 error 处理不一致

### 安全风险
- `file_tools.py` 写入路径限制依赖配置，容易配置错误
- `approval.py` 审批超时后默认行为需确认
- `delegate_tool.py` 子代理隔离程度需验证
- `~/.hermes/config.yaml` 写入需额外保护（已有 fix）
