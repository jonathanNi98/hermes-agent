## 二、学习顺序（基于工程依赖关系）

```
1. Agent Loop 总览         ← 理解"对话怎么跑"
2. CLI / Gateway 入口       ← 理解"怎么启动"
3. Context / Prompt Builder  ← 理解"模型看到什么"
4. Runtime / Provider       ← 理解"模型怎么被调用"
5. Tool Registry            ← 理解"工具怎么注册"
6. Tool Executor            ← 理解"工具怎么执行"
7. Memory / Session / State ← 理解"记忆怎么持久化"
8. Skills / Procedural Memory ← 理解"技能怎么被发现"
9. Delegation / Sub-agent / Cron ← 理解"任务怎么拆分"
10. Safety Boundary         ← 理解"危险操作怎么管控"
11. Observability / Logging  ← 理解"怎么追踪问题"
12. Evaluation / Tests      ← 理解"怎么验证行为"
13. Mini Hermes Day 1       ← 动手实现核心循环
14. Mini Hermes Day 2       ← 完善 + 演示
```

**为什么这样排？**
- 先搞懂"怎么跑"，再搞懂"跑的时候模型看到什么"，然后搞懂"底层依赖"
- 工具是模型的延伸，所以先有模型，再有工具
- 记忆是跨轮持久化的基础，放在主循环之后
- 安全和可观测性是工程保障，放在最后

## 四、Mini Hermes Agent 仿写项目

### 项目结构（已验证可行）

```
mini_hermes/
├── __init__.py
├── agent.py                  # AIAgent 主类
├── conversation_loop.py       # 主循环（从 run_agent.py 提取）
├── prompt_builder.py         # System prompt 构建
├── provider.py               # 模型调用抽象
├── tool_registry.py          # 工具注册（参考 tools/registry.py）
├── tool_executor.py          # 工具执行（参考 agent/tool_executor.py）
├── memory_manager.py         # 记忆管理（参考 agent/memory_manager.py）
├── memory_provider.py        # Provider 抽象（参考 agent/memory_provider.py）
├── safety.py                # 安全检查（参考 tools/path_security.py + approval.py）
├── trace_logger.py          # 日志追踪（参考 hermes_logging.py）
├── eval_runner.py           # 评估运行器（参考 batch_runner.py）
└── tools/
    ├── __init__.py
    ├── calculator.py         # 计算器
    ├── read_file.py         # 文件读取
    └── write_file.py        # 文件写入
```

### 核心实现要求

```
必须实现（最小可用）：
1. Agent Loop（while not done → API call → tool call → loop）
2. Tool Registry（register + dispatch）
3. Tool Executor（sequential execution）
4. Simple Memory（in-memory list，不需要 Provider 抽象）
5. Path Restriction（validate_within_dir）
6. JSONL Trace Logger（setup_logging 简化版）

选做（进阶）：
7. System Prompt 三层架构
8. Provider 抽象（支持 Anthropic / OpenAI）
9. Approval 状态机
10. Batch eval runner
```
